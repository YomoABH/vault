# Технический долг

## Сохранение заметок

### Race condition при параллельных writes в IndexedDB

**Файл:** `src/presentation/features/editor/components/NoteEditor.vue`

Если дебаунс сработал и save ещё in-flight, а пользователь успел снова изменить контент до завершения первого write — второй save читает устаревший `props.note` как базу. В редких случаях (большой контент, медленный write > 500ms) первый write может завершиться позже второго и перезаписать актуальные данные.

**Варианты решения:** очередь saves (каждый следующий ждёт завершения предыдущего) или отмена транзакции через `IDBTransaction.abort()`. Второй вариант — правильный слой для Phase 3, где появятся Web Locks.
